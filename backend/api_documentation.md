# ClearSkies Codebase Documentation

This document provides a comprehensive overview of the functions, classes, and methods across the ClearSkies backend codebase. It serves as an API and internal module reference.

---

## 1. FastAPI Endpoints (`main.py`)

The `main.py` file defines the RESTful API that the frontend and external services interact with.

*   `attribute_hotspot(hotspot: HotspotInputWithId)`: **POST `/api/attribution`**
    Accepts geospatial hotspot data, uses the `PollutionAttributionAgent` to predict the pollution source, optionally saves it to the database, and returns the prediction and confidence score.
*   `get_model_report()`: **GET `/api/attribution/model-report`**
    Returns the classification report from the `PollutionAttributionAgent`'s initial training phase.
*   `get_advisory(req: AdvisoryRequest)`: **POST `/api/advisory`**
    Takes a citizen's profile and forecast AQI, uses the `CitizenAdvisoryAgent` to generate a personalized health advisory, and optionally logs it to the database.
*   `get_ward_trends(ward: str)`: **GET `/api/trends/{ward}`**
    Fetches historical AQI for a specific ward and returns a `TrendReport` (averages, anomalies, festival spikes) generated by the `TrendAnalysisAgent`.
*   `get_all_trends()`: **GET `/api/trends`**
    Returns a summarized trend report for all wards in the system.
*   `get_heatmap(city: str, pollutant: str, resolution: int)`: **GET `/api/heatmap/{city}`**
    Fetches latest station readings and uses the `HeatmapAgent` to interpolate a dense grid of pollutant levels for map rendering.
*   `compare_cities(cities: Optional[str], days: int)`: **GET `/api/compare`**
    Fetches historical AQI and intervention data, then uses the `MultiCityComparisonAgent` to rank and compare cities based on their enforcement effectiveness.
*   `check_station_for_spike(req: StationCheckRequest)`: **POST `/api/emergency/check`**
    Checks a single station's recent readings for statistical spikes or severe thresholds using the `EmergencyDetectionAgent`.
*   `scan_city_for_spikes(city: str)`: **GET `/api/emergency/scan/{city}`**
    Checks every active station in a given city simultaneously for emergency conditions.
*   `chat(req: ChatRequest)`: **POST `/api/chat`**
    Takes a user query and optional live context, then uses the `ChatAssistantAgent` to retrieve relevant documents and synthesize an answer.
*   `dispatch_alert(req: AlertDispatchRequest)`: **POST `/api/alerts/dispatch`**
    Uses the `RealTimeAlertAgent` to dispatch a notification (push/SMS) based on the risk level and logs the dispatch to the database.
*   `get_alert_feed(limit: int)`: **GET `/api/alerts/feed`**
    Retrieves a list of recently dispatched alerts for the citizen-facing notification feed.
*   `get_analytics_summary(city: Optional[str])`: **GET `/api/analytics/summary`**
    Aggregates enforcement status, intervention outcomes, and source trends using the `AnalyticsAgent` for the Admin dashboard.
*   `get_forecast(ward: str, horizon_hr: int)`: **GET `/api/forecast/{ward}`**
    Uses the `AQIPredictionAgent` to forecast the AQI for a specific ward and time horizon based on current weather and traffic conditions.
*   `get_multi_horizon_forecast(ward: str)`: **GET `/api/forecast/{ward}/multi-horizon`**
    Returns a list of forecasts (24h, 48h, 72h) for a specific ward.
*   `get_forecast_model_report()`: **GET `/api/forecast/model-report`**
    Returns the regression metrics (MAE, RMSE) from the `AQIPredictionAgent`'s training phase.
*   `get_recommendations(req: RecommendationRequest)`: **POST `/api/recommendations`**
    Takes a ward, source, and forecast AQI, and uses the `RecommendationAgent` to output a list of concrete, role-based intervention actions.
*   `get_recommendations_from_pipeline(ward: str)`: **GET `/api/recommendations/{ward}`**
    A convenience endpoint that chains the attribution and prediction agents together to automatically generate recommendations for a ward.
*   `get_enforcement_queue(top_n: int)`: **GET `/api/enforcement/queue`**
    Uses the `EnforcementPrioritizationAgent` to rank known emission sources by risk score and return the top targets for inspection.
*   `check_emergency(ward: str, simulate_spike: bool)`: **GET `/api/emergency/check/{ward}`**
    A ward-level convenience wrapper that generates a synthetic station reading series for the ward and runs the station-keyed spike detector.
*   `check_all_emergencies()`: **GET `/api/emergency/check-all`**
    A convenience wrapper that generates synthetic stations for all wards (injecting a spike in Ward-5) and runs the emergency detector across all of them.

## 2. AI Agents (`agents/`)

### `advisory_agent.py`
*   `CitizenAdvisoryAgent.band_for_aqi(aqi: float) -> str`: Maps a numerical AQI value to a standard CPCB severity band.
*   `CitizenAdvisoryAgent.generate(profile: CitizenProfile, forecast_aqi: float) -> Advisory`: Combines a localized template and vulnerability addendums to create a personalized health advisory.
*   `CitizenAdvisoryAgent.generate_batch(profiles, ward_forecasts) -> list[Advisory]`: Generates advisories for a batch of user profiles.

### `alert_agent.py`
*   `RealTimeAlertAgent.should_notify(risk_level: str) -> bool`: Determines if a risk level is severe enough to warrant a push notification.
*   `RealTimeAlertAgent.dispatch_advisory(ward, risk_level, advisory_message, preferred_channel) -> Optional[AlertDispatch]`: Dispatches a routine notification if the risk band crosses the threshold.
*   `RealTimeAlertAgent.dispatch_emergency(ward, emergency_message, preferred_channel) -> AlertDispatch`: Dispatches an urgent, emergency-triggered alert (defaults to SMS).
*   `RealTimeAlertAgent.dispatch_bulk(ward_risk_map, message_by_ward) -> list[AlertDispatch]`: A batch entry point for dispatching notifications across multiple wards simultaneously.
*   `RealTimeAlertAgent._send(recipient, channel, message, risk_level) -> AlertDispatch`: Mock gateway call representing the actual Twilio/FCM dispatch.

### `analytics_agent.py`
*   `AnalyticsAgent.summarize(enforcement_df, interventions_df, source_df) -> AnalyticsSummary`: Aggregates pandas DataFrames of enforcement statuses, intervention ROIs, and source trends into a single dashboard-ready summary object.

### `attribution_agent.py`
*   `PollutionAttributionAgent.train(training_df, label_col) -> str`: Trains the RandomForestClassifier on historical labeled hotspot data and returns a classification report string.
*   `PollutionAttributionAgent.attribute(hotspot_row: pd.Series) -> AttributionResult`: Predicts the source of a pollution hotspot (and calculates confidence) based on input features.
*   `PollutionAttributionAgent.attribute_batch(hotspot_df) -> list[AttributionResult]`: Runs `attribute()` over a DataFrame of hotspots.
*   `PollutionAttributionAgent._build_evidence(row, predicted_source) -> dict`: Extracts the specific feature (e.g., thermal anomaly count) that provides human-readable evidence for the prediction.

### `chat_agent.py`
*   `ChatAssistantAgent.retrieve(query: str) -> list[dict]`: Uses TF-IDF and Cosine Similarity to find the most relevant chunks from the knowledge corpus.
*   `ChatAssistantAgent.answer(query: str, live_context: Optional[dict]) -> ChatAnswer`: Retrieves context and synthesizes a templated answer, optionally injecting live ward data.
*   `ChatAssistantAgent._synthesize_answer(query, retrieved, live_context) -> str`: Stitches together the retrieved text chunks and live context into the final response string.

### `comparison_agent.py`
*   `MultiCityComparisonAgent.compare(history_df, source_df) -> list[CityComparisonEntry]`: Groups historical AQI and intervention data by city, calculates averages, and returns a sorted list ranking cities by effectiveness.

### `emergency_agent.py`
*   `EmergencyDetectionAgent.check_station(reading_series: pd.DataFrame) -> Optional[EmergencyAlert]`: Analyzes a station's time-series data for three triggers: statistical PM2.5 spikes, rapid AQI rate-of-change, or absolute severe AQI thresholds.
*   `EmergencyDetectionAgent.check_all_stations(readings_by_station: dict) -> list`: Iterates over a dictionary of station readings and returns all triggered alerts.
*   `EmergencyDetectionAgent._build_message(station_id, triggers, pm25, aqi, zscore_val, aqi_delta, window_minutes) -> str`: Formats the human-readable emergency message detailing exactly why the alert triggered.

### `enforcement_agent.py`
*   `EnforcementPrioritizationAgent._proximity_score(distance_km: float) -> float`: Calculates a normalized score based on how close an emission source is to a hotspot.
*   `EnforcementPrioritizationAgent._inspection_gap_score(days_since: int) -> float`: Calculates a normalized score based on how long it has been since a source was inspected.
*   `EnforcementPrioritizationAgent.score_source(row, ward_severity, hotspot_confidence) -> PrioritizedSource`: Computes the final composite risk score for a single emission source based on proximity, permit status, inspection gap, and severity.
*   `EnforcementPrioritizationAgent.prioritize(sources_df, ward_severity_map, hotspot_confidence_map) -> list[PrioritizedSource]`: Scores and ranks a DataFrame of emission sources, returning the prioritized queue.

### `heatmap_agent.py`
*   `HeatmapAgent.interpolate(station_df, city, pollutant, bbox) -> HeatmapGrid`: Uses Inverse Distance Weighting (IDW) to interpolate scattered station point readings into a dense, continuous grid of values.
*   `HeatmapAgent._infer_bbox(station_df, padding) -> dict`: Calculates the bounding box (north, south, east, west) based on the min/max coordinates of the provided stations.

### `prediction_agent.py`
*   `AQIPredictionAgent.train(training_df, target_col) -> dict`: Fits the underlying model (LGBMRegressor or GradientBoostingRegressor fallback) on historical data and calculates residual standard deviation for confidence intervals.
*   `AQIPredictionAgent.predict(current_conditions: dict, horizon_hr: int) -> ForecastResult`: Generates a point estimate and confidence interval (lower/upper bounds) for a specific time horizon.
*   `AQIPredictionAgent.predict_multi_horizon(current_conditions, horizons) -> list[ForecastResult]`: Returns predictions for multiple horizons (e.g., 24, 48, 72 hours).

### `recommendation_agent.py`
*   `RecommendationAgent.severity_band(aqi: float) -> str`: Maps an AQI value to its corresponding severity category.
*   `RecommendationAgent.recommend(ward, source, forecast_aqi, festival_context) -> RecommendationBundle`: Looks up the appropriate, rule-based intervention actions from the `ACTION_MATRIX` based on the pollution source and forecast severity.
*   `RecommendationAgent.recommend_from_attribution_and_forecast(attribution_result, forecast_result, festival_context) -> RecommendationBundle`: A convenience wrapper that accepts the direct object outputs from the Attribution and Prediction agents.

### `trend_agent.py`
*   `TrendAnalysisAgent.analyze_ward(df, ward, festival_calendar) -> TrendReport`: Calculates averages (weekday, weekend, monthly) and uses rolling z-scores to identify anomaly days within a ward's historical data.
*   `TrendAnalysisAgent.analyze_all_wards(df, festival_calendar) -> dict`: Runs `analyze_ward` across all unique wards in the provided DataFrame.
*   `TrendAnalysisAgent._detect_festival_spikes(ward_df, festival_calendar, window_days) -> list`: Compares AQI during specified festival windows against the preceding 30-day baseline to detect significant percentage increases.

## 3. Data Generators & Ingestion (`data/`, `pipeline/`)

### `mock_data.py`
*   `generate_hotspot_features(n_per_ward)`: Generates synthetic hotspot data with features like traffic density and thermal anomalies.
*   `generate_citizen_profile(user_id)`: Generates a mock user profile with specific vulnerabilities (e.g., asthma).
*   `generate_prediction_training_data(days)`: Creates historical rows with weather, traffic, and lag features to train the prediction agent.
*   `generate_current_conditions(ward)`: Creates a mock snapshot of current weather/traffic for inference.
*   `generate_emission_sources(n)`: Creates a list of registered emission sites (factories, construction) with permit statuses.
*   `generate_realtime_readings(ward, spike)`: *(Legacy)* Generates a ward-level time series of recent AQI readings.
*   `generate_station_snapshot(n_stations_per_ward)`: Generates a snapshot of the latest readings for all stations, used for heatmaps.
*   `generate_multi_city_history(days)`: Generates daily average AQI and intervention logs for multiple cities.
*   `generate_source_breakdown_by_city()`: Generates synthetic counts of hotspot sources per city.
*   `generate_station_reading_series(station_id, hours, spike)`: Generates an hourly time series for a single station, optionally injecting a sudden spike for the emergency detector.
*   `generate_alert_feed(n)`: Generates a mock history of dispatched push/SMS notifications.
*   `generate_enforcement_queue_snapshot(n)`: Generates a mock status breakdown of the current enforcement queue.
*   `generate_intervention_roi_series(days)`: Generates a timeseries of actions taken and the corresponding AQI drop.
*   `generate_historical_aqi(days)`: Generates daily AQI time series per ward with seasonal and festival patterns.

### `overpass_pull.py`
*   `fetch_osm_features(bbox, amenity_type, max_retries, timeout) -> gpd.GeoDataFrame`: Queries the OpenStreetMap Overpass API for polygons representing industrial zones, construction sites, or highways, and returns a GeoPandas DataFrame. Includes retry and backoff logic for rate limits.

## 4. Database Access (`db/`)

### `database.py`
*   `get_engine()`: Lazily initializes and returns the SQLAlchemy Engine connected to the Neon Postgres database.
*   `get_session_factory()`: Returns a `sessionmaker` bound to the engine.
*   `get_session()`: A context manager for safely acquiring and committing/rolling back a database session.
*   `is_db_configured() -> bool`: Checks if the `DATABASE_URL` environment variable is set.

### `repository.py`
*(All functions execute raw SQL queries via SQLAlchemy `text()` and return pandas DataFrames or scalar values)*
*   `fetch_hotspot_features()`: Retrieves hotspot feature rows (traffic, construction density, etc.).
*   `fetch_historical_aqi()`: Aggregates station readings to a daily average per ward.
*   `fetch_citizen_profile()`: Retrieves a citizen's vulnerability profile.
*   `fetch_latest_forecast()`: Retrieves the latest predicted AQI for a ward.
*   `fetch_active_hotspots_for_map()`: Retrieves hotspots and extracts lat/lon from the PostGIS geometry column.
*   `insert_hotspot_attribution()`: Updates a hotspot record with the predicted source and confidence score.
*   `fetch_latest_station_readings()`: Retrieves the single latest reading for every active station.
*   `fetch_multi_city_summary()`: Retrieves daily average AQI grouped by city.
*   `fetch_intervention_summary()`: Retrieves intervention counts and average AQI drops grouped by city.
*   `fetch_source_breakdown_by_city()`: Retrieves counts of attributed hotspot sources grouped by city.
*   `fetch_recent_readings()`: Retrieves the recent hourly time series for a specific station.
*   `fetch_all_active_station_ids()`: Retrieves a list of all active station IDs (used by the emergency batch scanner).
*   `insert_alert_log()`: Inserts a record of a dispatched notification into the audit log.
*   `fetch_recent_alerts()`: Retrieves the latest dispatched alerts.
*   `fetch_enforcement_status_counts()`: Retrieves a count of enforcement queue items by status (pending/assigned/resolved).
*   `fetch_intervention_roi_timeseries()`: Retrieves the timeline of recorded interventions and their AQI outcomes.
*   `insert_advisory()`: Inserts a record of a dispatched health advisory.

### `run_migration.py`
*   `main()`: A standalone script that reads a `.sql` file path from `sys.argv` and executes it directly against the Neon database using `psycopg2`.
